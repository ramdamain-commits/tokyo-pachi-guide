# 東京パチ・スロ特定日ガイド

東京都内の主要エリアのパチンコ・パチスロ店について、旧イベント日（特定日）パターンと店舗概要をローカルで即答できるようにする個人用ガイド。「今日どの店が該当日か」を毎回AIに聞かずに済ませるのが目的。

**公開方針**: 2026-07-17 に非公開→公開へ変更（本人判断・当初の「非公開」記述を上書き）。GitHub 公開リポジトリ＋GitHub Pages でビューワを公開し、portal からも参照する。掲載データは店舗の公開情報・第三者サイトの過去傾向推定のみで、個人情報（居住エリア・実名等）は含めない。旧イベント日は推定であり店舗の公式情報ではない点に留意（下記「注意」参照）。

## 構成

- `data/stores.json` — 店舗マスタ（エリア・台数・特定日ルール・抽選方式・閉店時刻・換金率・概要・出典・確認日）。2026-07-20 に下北沢を追加し全35店（上野4/秋葉原5/新宿6/渋谷4/下北沢3/池袋8/新橋3/京橋銀座2）
- `data/media.json` — 取材媒体マスタ（媒体名・取材傾向・対象ジャンル・確度・公式スケジュールURL・出典）。2026-07-20 に17媒体へ拡充。店舗単位ではなく媒体単位の一般的傾向（伝聞情報、公式公約ではない）
- `data/coverage.json` — 店舗×媒体の取材実績（storeId・mediaName・頻度・直近確認月・確度・出典）。過去実績に基づく推定・伝聞であり公式情報ではない。掲載が無い＝実績なし、ではない（欠測）
- `match.mjs` — 日付マッチングロジック本体（`rules[].type` 全7種の判定・`ruleLabel` によるバッジ用日本語ラベル生成）
- `coverage.mjs` — coverage.json の参照整合検証と店舗/媒体からの結合ロジック
- `index.html` / `app.js` — ビューワ（今日/明日/日付指定/エリア別一覧/取材傾向の5タブ）。店舗カードに取材実績バッジ、媒体カードに実績店一覧と公式スケジュールリンクを表示
- `test/` — `node --test test/*.test.mjs` で実行（match 18件+coverage 5件）
- `docs/取材予定の確認手順.md` — 「今週どこに取材が入るか」を調べる巡回手順とリンク集
- `docs/superpowers/specs/` — 設計書

## 閲覧方法

公開ビューワ: **https://ramdamain-commits.github.io/tokyo-pachi-guide/**

ローカルで動かす場合、ES module + fetch を使うため `file://` では動作しない。リポジトリルートで以下を実行してブラウザから `http://localhost:8099/` を開く。

```
python3 -m http.server 8099
```

## 対象エリア

上野・秋葉原・新宿・渋谷・下北沢・池袋・新橋・京橋/銀座近辺（8エリア、全35店）

## 注意

旧イベント日は第三者サイトの過去傾向推定であり、店舗の公式情報ではない（風営法上、設定示唆の公式イベントは存在しない）。取材実績（coverage.json）も過去実績に基づく推定・伝聞であり、店舗・媒体の公式情報ではない。データには必ず出典URLと確認日を付け、四半期に1回棚卸しする。

## 更新方法

特定日はパターンルール（「5のつく日」等）のため原則更新不要。店舗の閉店・リニューアル・傾向変化があれば `data/stores.json` を修正する。`data/coverage.json`（取材実績）は鮮度が落ちやすいため、stores.json と同じ四半期棚卸しの対象とする。媒体の追加・スケジュールURL変更は `data/media.json` を先に直し、`docs/取材予定の確認手順.md` のリンク集を追随させる。
