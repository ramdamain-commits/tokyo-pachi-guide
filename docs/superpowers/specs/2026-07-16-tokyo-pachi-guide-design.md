# 東京パチ・スロ特定日ガイド 設計書（2026-07-16）

## 背景・目的

claude.ai で都度「今日、京橋/都内から行ける特定日該当のパチンコ店はどこか」を調べていた（旧イベ日×店舗×抽選方式×立ち回りの突合）。この問い合わせは、

- 旧イベント日が「5のつく日」「毎月11日」「ゾロ目」等の**静的パターンルール**であること
- 店舗概要（台数・抽選方式・閉店時刻）が低頻度でしか変わらないこと

から、一度データ化すれば日付マッチングだけで毎日答えが出る。本PJはこれをローカルの静的ビューワとしてPJ化し、AIへの都度問い合わせをなくす。

## 確定方針（2026-07-16 ユーザー承認済み）

1. **対象エリア**: 上野・秋葉原・新宿・新橋・池袋・渋谷・京橋/銀座近辺の7エリア。各エリア主要4〜6店、計30〜40店規模
2. **スコープ**: 旧イベ日パターン＋店舗概要のみ。取材・来店イベント（月次変動）はv2送り
3. **公開方針**: ローカルgitのみ。GitHubなし・portal/life-ops-export掲載なし（ギャンブル関連＋行動圏情報のため）

## データソース

| 優先 | ソース | 用途 |
|------|--------|------|
| 第1 | すろざんまい 東京都まとめ（manmai.club/?p=5615） | 特定日パターンの収集元（日付別・数百店網羅・2026-06-25更新確認済み） |
| 第2 | みんパチ（minpachi.com 店舗別ページ） | 台数・換金率・住所等の**店舗詳細の補完専用**（パターン突合には使わない） |

制約: 旧イベ日は第三者の過去傾向推定であり公式情報ではない。全レコードに出典URLと確認日を必須とし、AI推測での補完は禁止（出典なしイベントは記載しない）。

## 成果物

### 1. `data/stores.json`（店舗マスタ）

```jsonc
{
  "updated": "2026-07-16",
  "stores": [
    {
      "id": "espace-ueno-honkan",
      "name": "エスパス日拓上野本館",
      "area": "上野",                    // 7エリアのいずれか
      "address": "東京都台東区上野4-8-6", // みんパチ由来。不明はnull
      "access": "上野広小路駅 徒歩3分",
      "slots": 513,                      // スロ台数（不明はnull）
      "pachinko": 250,                   // パチ台数（不明はnull）
      "rules": [                         // 特定日ルール（構造化・複数可）
        { "type": "digit", "value": 5 },             // 5のつく日
        { "type": "digit", "value": 7 },             // 7のつく日
        { "type": "monthday", "value": 21, "label": "月周年" }, // 毎月21日（毎月開催の月周年）
        { "type": "annual", "month": 3, "day": 21, "label": "周年" }, // 年1回の周年（開店月日固定）
        { "type": "zorome" },                        // 日ゾロ目(11,22)
        { "type": "month-day-match" },               // 月日重なり(7/7等)
        { "type": "weekday", "value": 3 },           // 毎週水曜（getDay()準拠: 日=0〜土=6）
        { "type": "nth-weekday", "nth": 1, "weekday": 6 }    // 第1土曜（weekdayはgetDay()準拠）
      ],
      "lottery": { "method": "アプリ抽選", "draw": "9:30", "auth": "9:45", "lineup": "9:50" },
      "open": "10:00", "close": "23:00",
      "exchange": "非等価(47枚/53枚)",   // 不明はnull
      "memo": "スロ大型店。5のつく日はジャグラー寄り等の傾向メモ",
      "sources": [ { "url": "https://...", "checkedAt": "2026-07-16" } ]
    }
  ]
}
```

`rules[].type` の全種別: `digit`（○のつく日）/ `monthday`（毎月○日）/ `annual`（年1回・月日固定。年周年等）/ `weekday`（毎週○曜）/ `nth-weekday`（第n○曜）/ `zorome`（11日・22日）/ `month-day-match`（月と日が同数）。これで すろざんまい の分類（○の付く日系・毎月○日系・毎週○曜系・第○△曜系・ゾロ目系・月日重なり）＋年周年を全カバーする。

マッチング仕様の明確化:
- `monthday` は `value: "last"` で月末日（月により28〜31日）を表す。`value: 31` 等が存在しない月は単に該当なし（繰り上げしない）
- 曜日番号は JS `Date.getDay()` 準拠（日=0〜土=6）
- 「今日」の判定はローカル(JST)基準。日付生成は `new Date(y, m-1, d)` を使い UTC文字列パースによるズレを避ける（tokyo-solo-guide app.js と同方針）
- 1つの日付が複数ルールにマッチした場合（例: 7/7 は digit:7 と month-day-match）は、マッチした全ルールをバッジ表示する（統合しない）

### 2. ビューワ（`index.html` + `app.js`）

tokyo-solo-guide の実装パターンを流用（静的HTML+fetch+描画、フレームワークなし）。

- **今日/明日/日付指定タブ**: 指定日に `rules` がマッチする店を該当ルール名付き（複数マッチは全バッジ）でエリア順に表示。「該当なしの日」はその旨表示
- **エリア別一覧タブ**: 店カード（台数・特定日一覧・抽選方式・閉店・換金率・メモ・出典リンク・確認日）
- **固定注記**: 画面上部に「特定日は第三者サイト由来の過去傾向推定であり公式情報ではない」旨を常設表示
- 閲覧は `python3 -m http.server`（file:// はブラウザMCP制約により不可）

### 3. 検証（`test/match.test.mjs`）

日付マッチング関数を `app.js` から分離した `match.mjs` に置き、Node の `node --test` でユニットテスト（wikiベース記述にスクリプト検証を挟むルール準拠）。最低限含める境界ケース:

- `monthday: 31` を含まない月（2月・4月等）で該当なしになること／`"last"` が各月の月末日に正しくマッチすること
- うるう年 2/29（`monthday`・`annual` とも）
- `zorome` と `monthday: 11` を併記した店で両ルールが同時マッチすること（重複排除しないこと）
- `month-day-match`（7/7等）と `digit` の同時マッチ
- 年跨ぎ（12/31→1/1）と `nth-weekday` の月初判定

収集データの突合は「特定日パターン＝すろざんまい×アナスロ（食い違い店＋各エリア上位2店）」「店舗詳細＝みんパチで補完」の役割分担で行い、ソース間の食い違いは memo に併記する。

## 収集手順（次セッション）

1. researcher サブエージェント3並列（Sonnet）でエリア分担収集、`_staging/pachi-{担当}.md` へ直書き
   - ①上野＋秋葉原 ②新宿＋渋谷 ③池袋＋新橋＋京橋/銀座
   - 指示に含める: 出典URL必須／推測禁止／店舗数下限（各エリア4店）／特定日パターンはすろざんまいを第1ソースとし、みんパチは店舗詳細（住所・台数・換金率）の補完に使う／みんパチにパターン記載がある場合は食い違いの有無も報告する
2. メイン側でアナスロ照合（Playwright MCP経由。researcher は Playwright を持たないためメイン作業）: パターン表記が食い違った店＋各エリア上位2店を対象に三点照合 → 統合・矛盾解決 → `data/stores.json` 生成
3. implementer にビューワ実装を委任（変更対象ファイルを明示列挙）
4. マッチングテスト実行＋ビューワをhttp.server経由で表示確認

## 鮮度管理

- 全レコードに `checkedAt`。四半期に1回（10月・1月…）閉店/リニューアル/傾向変化を棚卸しし、TASKS.md に定期タスクとして記録
- 特定日パターン自体は原則更新不要（変わるのは店の存廃と傾向メモ）

## 非スコープ（v2以降の候補）

- 取材・来店イベント（月次変動、ホールナビ取込）
- 出玉データ（アナスロ等）の自動取得・後ヅモ支援
- 名古屋等の他都市エリア追加
